## AstrBot 上下文工程分析

对 AstrBot 项目进行了完整的上下文工程分析，结合"上下文五层结构"理论框架（任务/知识/工具/环境/记忆），深度审读了 30+ 核心模块文件，输出了一份详细的分析文档。

关键文件：
- 上下文装配核心：`astrbot/core/astr_main_agent.py`（build_main_agent，1300+ 行）
- Agent 循环引擎：`astrbot/core/agent/runners/tool_loop_agent_runner.py`
- 上下文压缩：`astrbot/core/agent/context/` 子包
- 多 Agent 协同：`astrbot/core/subagent_orchestrator.py` + `astrbot/core/agent/handoff.py`
- 知识库：`astrbot/core/knowledge_base/kb_mgr.py`
- 工具管理：`astrbot/core/provider/func_tool_manager.py`

核心发现：知识层和工具层工程化程度高，任务层和记忆层较薄弱，Context Engine 缺乏独立抽象。
